Magento REST API


Jednostavno objašnjeno, REST (Representational State Transfer) API definiše skup funkcija na kojima programeri mogu da obavljaju zahteve i dobijaju odgovore (response). Interakcija se vrši preko HTTP protokola. Prednost ovakvog pristupa je široka upotreba HTTP protokola, i iz tog razloga REST API može praktično da se koristi za svaki programski jezik. Drugi Magentov API je SOAP.

Značajne karakteristike REST API ukoliko je host na Magentohost-u:

  • Resursima se pristupa slanjem HTTP zahteva Magento API serveru. Server vraća odgovor koji sadrži podatke koje ste tražili ili status indikatora (često vrati i oboje).
  • Svi resursi se nalaze na adresi.
  • Svi resursi mogu vratiti različite HTTP statuse kodova.
  • Ukoliko zahtevate poseban odgovor, morate dodati određenu putanju za bazni URL koji određuje resurs.

Magento REST API omogućava upravljanjem velikim brojem funckcija, među kojima su:

  • Upravljanje kupcima
  • Upravljanje adresama kupaca
  • Upravljanje proizvodima
  • Preuzimanje narudžbina kupaca
  • Upravljanje zalihama
Korišćenje i pristup ovakvim podacima, REST API uspeva korišćenjem HTTP glagola. HTTP glagoli se koriste za upravljanje stanjem resursa. REST API koristi 4 osnovna glagola, a to su GET, POST, PUT i DELETE. Podatke možete dobiti korišćenjem HTTP GET, kreirati preko HTTP PUT ili HTTP POST, a obrisati pomoću HTTP DELETE.

Na sledećoj slici možete videti algoritam rada REST API-a.
